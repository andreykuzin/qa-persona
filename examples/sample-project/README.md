# Sample project — listmonk

**Upstream:** https://github.com/knadh/listmonk — self-hosted newsletter and mailing-list manager (~93 Go files, ~57 Vue/TS files).

This directory shows what qa-persona produces when run against a real open-source codebase. The dogfood agent ran Phases 1–3 (value loop, personas, scenario catalog) against listmonk on 2026-05-12; see [the full dogfood report](https://github.com/andreykuzin/qa-persona/issues/1) for timings, confusion points, and suggested skill improvements.

## The value loop

listmonk closes one loop: an admin creates a mailing list, adds or imports subscribers, authors a campaign, and sends it — and subscribers receive the email, click its tracked links, and manage their own subscriptions via self-service public pages reached through UUID-keyed links embedded in every message. The loop has two distinct actors (admin and subscriber) that never share a session: admins authenticate via password or OIDC; subscribers are anonymous and identified only by a server-generated UUID stored in the URL. Artifacts flowing through the loop are **Lists** (public/private, single/double opt-in), **Subscribers** (with per-list subscription states: unconfirmed → confirmed → unsubscribed → blocklisted), and **Campaigns** (draft → scheduled → running → paused → finished/cancelled). The loop re-opens after each finished campaign because bounces trigger subscriber-status changes that affect the next campaign's eligible recipient set.
